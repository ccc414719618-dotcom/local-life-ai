### 技术问题与修复记录

#### 1. yfinance MultiIndex 列格式问题（严重）
- **现象**：`df['Close']` 返回 MultiIndex DataFrame（列名 × ticker），不是普通 Series
- **具体错误**：`close.iloc[-1]` 是 `Series`，不是标量 → `float()` 报错；`.tolist()` 在单列 DataFrame 上不适用
- **调试输出**：
  ```
  df.columns: MultiIndex([('Close', 'BTC-USD'), ('High', 'BTC-USD'), ...])
  close.iloc[-1]: Series with name='BTC-USD', dtype=float64
  ```
- **修复方案**：统一用 `float(df[('Close', ticker)].iloc[-1])` 取单值

#### 2. 文件截断问题（危险）
- **原因**：用 `write` 重写 `get_data` 时，整文件被截断为只有 `get_data` 函数
- **教训**：`get_data` 重写后，后面所有函数（`get_ohlcv`, `safe_download`, `generate_charts`, `generate_ppt`, `build_doc_markdown`, `create_feishu_doc`, `send_summary`）全部丢失
- **修复**：从 git 恢复完整原始文件，然后只替换 `get_data` 函数体

#### 3. `ModuleNotFoundError: No module named 'finance'`
- **原因**：`daily_push.py` 从 `finance/` 子目录运行，`import finance.data_fetcher` 找不到父包
- **修复**：文件顶部加 `sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))`

#### 4. `safe_download` 函数丢失
- **原因**：被上面的文件截断问题一起丢了
- **修复**：手动加回（此函数在早期对话中添加，未进入 git）

#### 5. `generate_ppt` 未定义
- **原因**：`daily_push.py` 里调用了 `generate_ppt(data)`，但该函数定义在 `pptx_generator.py` 中
- **修复**：`daily_push.py` 顶部加 `from pptx_generator import generate_ppt`

#### 6. `data` 结构不匹配（PPT 期望嵌套格式）
- **PPT 期望**：`data['crypto']['bitcoin']['price']`
- **get_data 输出**：`data['bitcoin']`（顶层）
- **修复**：在 `main()` 里加格式转换：
  ```python
  adapted = {
      'crypto': {'bitcoin': data['bitcoin'], 'ethereum': data['ethereum'], 'solana': data['solana']},
      'metals': {'gold': data['gold'], 'silver': data['silver']},
      'indices': {k: data[k] for k in ['sp500', 'dowjones', 'nasdaq', 'ftse100', 'nikkei225', 'sse', 'hsi', 'dax']},
      'fear_greed': data['fear_greed'],
      'btc_spy_corr': data['btc_spy_corr'],
  }
  ```

#### 7. yfinance Rate Limit（持续性问题）
- **现象**：`YFRateLimitError: Too Many Requests`，大量 ticker 请求失败
- **已采取措施**：
  - 类级节流 `_ticker_last_request`：每个 ticker 间隔 ≥ 2.5s
  - 每次请求后等 15s（`safe_download` 默认 delay=15）
  - 4 次重试 + (attempt+1)*10s 指数退避
- **状态**：基本缓解，数据下载全部成功（101s 完成 9 个数据模块）
